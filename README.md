## 1. Информация о виртуальной машине

## 2. Архитектура Docker Compose проекта
Проект развернут с помощью Docker Compose (Compose v2) и состоит из трех изолированных контейнеров:
1. **proxy (Nginx alpine):** Выступает в роли Reverse Proxy, принимает входящие запросы на порту 80 хоста и перенаправляет их на контейнер с приложением
2. **app (traefik/whoami):** Простая веб-служба, возвращающая информацию об HTTP-запросе. Порты наружу не публикуются (доступен только через прокси)
3. **db (PostgreSQL 15 alpine):** Официальный образ базы данных. Настроена политика перезапуска, проверка здоровья (healthcheck) и персистентное хранение данных

**Особенности конфигурации безопасности:**
* Все параметры подключения к БД (пользователь, имя базы данных) и секреты (пароль) полностью вынесены в изолированный файл переменных окружения `.env`. В основном манифесте `docker-compose.yml` открытые пароли отсутствуют
* Все контейнеры объединены в кастомную изолированную сеть типа bridge (`app_network`)
